# OpenAI-Assisted Project Descriptions

## Overview

The system automatically generates professional project descriptions using OpenAI when GitHub descriptions are missing, incomplete, or flagged for regeneration.

## How It Works

### Automatic Updates

When a new repository is added to your portfolio (via whitelist or `portfolio` topic), the system will:

1. **Title Formatting**: Automatically format repository names
   - `Discord-Giveaway-BOT` â†’ `Discord Giveaway BOT`
   - `myhub` â†’ `Myhub`
   - Hyphens are converted to spaces, words are capitalized

2. **Description Generation**: 
   - If GitHub description is missing/empty â†’ Generate with OpenAI
   - If description is placeholder text â†’ Generate with OpenAI
   - Uses repository name, languages, topics, and README content as context
   - Generates 2-4 sentences in UK English, professional tone

3. **Language Detection**:
   - Automatically detects languages from GitHub's language statistics
   - Sorted by code volume (most used first)
   - Formatted nicely (e.g., `TypeScript`, `JavaScript`, `Python`)

### Example: Discord-Giveaway-BOT

When `Discord-Giveaway-BOT` is processed:

**Input:**
- Repository name: `Discord-Giveaway-BOT`
- GitHub description: (empty or placeholder)
- Languages: `{ TypeScript: 5000, JavaScript: 2000 }`
- Topics: `['discord', 'bot', 'giveaway']`

**Output:**
- **Title**: `Discord Giveaway BOT` (formatted)
- **Description**: Generated by OpenAI based on code analysis
- **Tech Stack**: `TypeScript`, `JavaScript` (from language detection)

### Caching

All generated descriptions are cached in PostgreSQL (`project_descriptions` table):
- Prevents unnecessary API calls
- Ensures idempotency (same result on repeated runs)
- Fast retrieval on subsequent requests

### When Descriptions Are Generated

Descriptions are **only** generated when:
- âś… GitHub description is missing or empty
- âś… Description is placeholder text (e.g., "A project", "Repository")
- âś… Repository is explicitly flagged for regeneration

Descriptions are **NOT** generated when:
- âťŚ GitHub description exists and is valid
- âťŚ Description is already cached (unless flagged for regeneration)

### Manual Regeneration

To force regeneration of a description, you can flag it in the database:

```sql
UPDATE project_descriptions 
SET regenerate_flag = TRUE 
WHERE repo_name = 'Discord-Giveaway-BOT';
```

The next sync will regenerate the description.

## Configuration

### Environment Variables

```bash
OPENAI_API_KEY=sk-...  # Required for description generation
GITHUB_TOKEN=...        # Optional, increases rate limits
GITHUB_USERNAME=...     # Your GitHub username
```

### Database Schema

The `project_descriptions` table stores:
- `repo_name` - Repository identifier (primary key)
- `description` - Generated or cached description
- `is_auto_generated` - Whether description was AI-generated
- `source` - 'openai' or 'github'
- `regenerate_flag` - Flag to force regeneration
- `created_at` / `updated_at` - Timestamps

## Troubleshooting

### Descriptions Not Generating

1. **Check API Key**: Verify `OPENAI_API_KEY` is set in `.env`
2. **Check Logs**: Look for `[OpenAI]` messages in server logs
3. **Check Database**: Verify `project_descriptions` table exists
4. **Check Credits**: Ensure OpenAI account has API credits

### Description Quality

If generated descriptions need improvement:
- The system uses repository structure, languages, topics, and README
- Better README content = better descriptions
- More specific topics = better context

## API Usage
